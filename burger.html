Burger App (Go Burger OR Go Home)

As a full-stack web applications scale, we needed to address several issues related to that growth, such as, organization of the codebase, ongoing development, maintenance and continued collaboration. My previous applications were easily managed as they only consist of a handful of files. I could limit the code base to these few files, but as I add functionality to my applications, each of those files grew in the length of lines of code composed of unnecessarily complicated logic. Imagine how difficult it would be to resolve merge conflicts!

In this application, I modularize my application following the MVC paradigm. The Model View Controller is an architectural pattern that structures my codebase in three distinct sections according to a software design philosophy known as the separation of concerns. I dynamically generate the HTML using a template engine. In implementing a template engine, I was able to separate the concern of client-side rendering from other aspects of the application. This is the View layer in the MVC framework.

In this assignment, I create a burger logger application with MySQL, Node, Express, Handlebars and a homemade ORM. I followed the MVC design pattern by using Node and MySQL to query and route data in my app. I also used Handlebars to generate my HTML. This application was deployed on Heroku.
